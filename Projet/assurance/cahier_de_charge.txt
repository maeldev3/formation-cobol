## 1. INTRODUCTION

### 1.1 Objectif du projet
Développer une application de gestion d'assurance en langage COBOL permettant de gérer les clients, les contrats, les paiements et les sinistres avec des fichiers séquentiels.

### 1.2 Contexte
Le projet vise à automatiser les opérations courantes d'une compagnie d'assurance :
- Gestion des clients (inscription, modification, suppression)
- Gestion des contrats (création, résiliation)
- Gestion des sinistres (déclaration, traitement, indemnisation)
- Gestion des paiements (échéances, encaissement)

### 1.3 Technologies utilisées
- **Langage** : COBOL (GnuCOBOL)
- **Stockage** : Fichiers séquentiels (.DAT)
- **Plateforme** : Linux Ubuntu
- **Compilateur** : cobc (GnuCOBOL)

---

## 2. COMPRENDRE LE CAHIER DE CHARGE

### Qu'est-ce qu'un cahier de charge ?
Un cahier de charge est un document qui décrit **CE QUE** le logiciel doit faire (pas COMMENT). C'est un contrat entre le client et le développeur.

### Pourquoi est-ce important ?
- ✅ Le client sait exactement ce qu'il va recevoir
- ✅ Le développeur sait exactement ce qu'il doit faire
- ✅ Évite les malentendus et les modifications en cours de route
- ✅ Sert de base pour les tests et la validation

### 4.2 Gestion des CONTRATS

#### 4.2.1 Structure du fichier CONTRATS.DAT


#### 4.2.2 Types de contrats
| Code | Type | Description | Taux prime |
|------|------|-------------|------------|
| AUTO | Automobile | Assurance véhicule | 3% |
| HABI | Habitation | Assurance maison | 2% |
| SANT | Santé | Assurance médicale | 5% |
| VIE | Vie | Assurance décès | 4% |

#### 4.2.3 Règles de gestion
- ID contrat unique : TYPE + 4 chiffres (AUTO0001)
- Un contrat est lié à un seul client
- Montant annuel = valeur assurée × taux prime
- Date début ≤ Date fin
- Contrat actif uniquement si paiements à jour

#### 4.2.4 Fonctions CONTRATS

| Fonction | Description | Règles |
|----------|-------------|--------|
| CREER_CONTRAT | Créer un nouveau contrat | Client doit exister, type valide |
| RESILIER_CONTRAT | Résilier un contrat | Sans sinistre en cours |
| LISTER_CONTRATS | Afficher contrats | Par client ou tous |
| RENOUVELER | Prolonger contrat | 1 an supplémentaire |

#### 4.4.1 Structure du fichier SINISTRES.DAT

#### 4.4.2 Statuts des sinistres
| Code | Statut | Description |
|------|--------|-------------|
| D | Déclaré | En attente de traitement |
| E | En instruction | Expertise en cours |
| A | Accepté | Indemnisation validée |
| R | Refusé | Non couvert |

#### 4.4.3 Règles de gestion
- ID sinistre : `SIN` + AAAAMMJJ + 2 chiffres
- Un sinistre est lié à un contrat
- Délai de déclaration : 5 jours ouvrés
- Franchise selon contrat : 50€ à 500€
